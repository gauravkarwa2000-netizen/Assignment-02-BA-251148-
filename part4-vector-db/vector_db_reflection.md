## Vector DB Use Case

A traditional keyword-based database search would not be sufficient for searching large legal contracts using natural language queries. This is because keyword search depends on exact word matching, and in legal documents, the same idea can be written in many different ways. For example, if a lawyer searches for “termination clauses,” the system might miss sections that use terms like “contract cancellation” or “end of agreement.” This makes keyword-based search unreliable and time-consuming, especially when dealing with long documents like 500-page contracts.

A vector database helps solve this problem by using semantic search. Instead of focusing on exact words, it converts both the documents and the query into embeddings, which are numerical representations of their meaning. These embeddings are stored in a vector database. When a user enters a query in plain English, the system converts it into an embedding and finds the most similar sections of the contract based on meaning, even if the wording is different.

This makes the search process much more efficient and accurate. Lawyers can quickly find relevant clauses without reading the entire document. Therefore, a vector database plays an important role in enabling intelligent and user-friendly search in such systems.
